Temat: Teoria dotycząca pętli w JavaScript

Rodzaje pętli

1. Pętla for

Nazwa pętli:

Pętla for

Konstrukcja:

for (zainicjowanie_zmiennej; warunek_kończący_wykonywanie_pętli; zmiana_zmiennej)
{
    instrukcje do wykonania;
}

Opis:

Pętla for jest stosowana wtedy, gdy z góry wiadomo, ile razy dany fragment kodu ma zostać wykonany. Najpierw następuje zainicjowanie zmiennej, następnie sprawdzany jest warunek. Jeśli warunek jest prawdziwy, wykonywane są instrukcje znajdujące się w pętli. Po wykonaniu instrukcji następuje zmiana zmiennej, na przykład zwiększenie jej o 1.

Przykład w JavaScript:

for (let i = 1; i <= 5; i++)
{
    document.write("To jest powtórzenie numer: " + i + "<br>");
}

2. Pętla while — z kontrolowanym wejściem

Nazwa pętli:

Pętla while

Konstrukcja:

while (warunek)
{
    instrukcje do wykonania;
}

Opis:

Pętla while jest pętlą z kontrolowanym wejściem. Oznacza to, że warunek sprawdzany jest przed wykonaniem instrukcji. Jeśli warunek jest prawdziwy, pętla wykona instrukcje. Jeśli warunek od początku jest fałszywy, pętla nie wykona się ani razu.

Przykład w JavaScript:

let i = 1;

while (i <= 5)
{
    document.write("Liczba: " + i + "<br>");
    i++;
}

3. Pętla do while — z kontrolowanym wyjściem

Nazwa pętli:

Pętla do while

Konstrukcja:

do
{
    instrukcje do wykonania;
}
while (warunek);

Opis:

Pętla do while jest pętlą z kontrolowanym wyjściem. Oznacza to, że najpierw wykonywane są instrukcje, a dopiero potem sprawdzany jest warunek. Dzięki temu pętla do while zawsze wykona się przynajmniej jeden raz, nawet wtedy, gdy warunek jest fałszywy.

Przykład w JavaScript:

let i = 1;

do
{
    document.write("Liczba: " + i + "<br>");
    i++;
}
while (i <= 5);
